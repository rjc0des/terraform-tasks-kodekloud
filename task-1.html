Task 1

Key Pair Lifecycle for the Nautilus Migration

Terraform controls RSA generation, cloud publication, and local storage

🧭

Problem Statement

Must Have

A controlled AWS migration requires a Terraform-owned RSA key pair so later phases can launch instances without manual steps. The key must be named devops-kp, use the RSA algorithm, land in /home/bob/devops-kp.pem, and everything must live inside /home/bob/terraform/main.tf.

The private key must never be handcrafted; Terraform needs to generate it and persist it locally so future steps rely on a known credential.

💡

Approach & Thinking

Strategy

Generate TLS key material, register the public key in AWS, then write the private key to disk. This isolates responsibilities and keeps the pipeline modular.

Alternative: run ssh-keygen manually. Downside: external dependencies break the self-contained requirement and hurt repeatability. Terraform-controlled generation is simpler, deterministic, and ready for rotation.

Using local_file ensures Terraform owns both cloud and controller artifacts, a must for this incremental migration.

🛠️

Terraform Solution

Blueprint 
resource "tls_private_key" "devops_key" {
  algorithm = "RSA"
  rsa_bits  = 4096
}

resource "aws_key_pair" "devops_kp" {
  key_name   = "devops-kp"
  public_key = tls_private_key.devops_key.public_key_openssh
}

resource "local_file" "save_to_local" {
  content         = tls_private_key.devops_key.private_key_pem
  filename        = "/home/bob/devops-kp.pem"
  file_permission = "0600"
}
▶️

Execution Flow

Steps
  1. terraform init in /home/bob/terraform to install the required providers.
  2. terraform plan to confirm the RSA key, AWS upload, and local file creation.
  3. terraform apply to run the plan, yield the key pair, and write the PEM file.

Ensure AWS credentials and write permissions exist before applying.

✔️

Why This Works

Proof

tls_private_key satisfies RSA and keeps Terraform in control.

aws_key_pair publishes the public key under the required name so cloud resources can consume it.

local_file writes the private key at the mandated path with secure permissions, closing the loop for future phases.

⚠️

Common Mistakes

Watch
  • Generating the key externally and only uploading the public half via Terraform.
  • Neglecting the local_file output and leaving the PEM only in state.
  • Missing file permission restrictions and exposing the private key.
  • Giving the key a different name or algorithm, breaking downstream expectations.
🪄

Final Thoughts

Wrap Up

Terraform owning both the RSA material and its AWS registration keeps the Nautilus migration phases consistent, auditable, and ready for the next bit-sized launch.
